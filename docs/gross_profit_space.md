# 毛利空间理解笔记

> 来源：用户“毛利空间”截图。当前为截图人工整理版，用于理解口径和业务含义；若后续拿到原始表格，需要以原表公式为准。

## 1. 这张表在回答什么问题

这张表不是某个商业模式的成本测算明细，而是在回答一个更上层的问题：

> **Robobus 业务的毛利空间，最多可以承受 Robovan 亏损多少？**

换句话说，它是在做一个“集团/业务组合视角”的约束：

```text
Robobus 毛利
≈ Robovan 可承受亏损上限
```

如果 Robovan 的亏损超过这个上限，就会侵蚀甚至吃掉 Robobus 的毛利。

截图中主要分成两块：

```text
Robobus
- 收入
- 成本
- 毛利
- 毛利率

Robovan
- 亏损上限
- 平台模式：
  - 单车亏损
  - 交付量
- 卖车模式：
  - 单车亏损
  - 交付量
```

|---|---|---:|---:|
| Robobus | 收入 | 50,000,000 | 50,000,000 |
| Robobus | 毛利 | 10,000,000 | 12,500,000 |

| Robovan | 亏损上限 | 10,000,000 | 12,500,000 |

截图里 Robovan 下方的“平台模式/卖车模式”的单车亏损和交付量暂时为空，说明这张表可能是一个待填的反推模板。

## 4. 这张表的核心逻辑

### Robobus 毛利计算

```text
Robobus 毛利 = Robobus 收入 × Robobus 毛利率
```

情景 A：

```text
50,000,000 × 20% = 10,000,000
```

情景 B：

```text
50,000,000 × 25% = 12,500,000
```

### Robovan 亏损上限

表里把 Robovan 的亏损上限设为等于 Robobus 毛利：

```text
Robovan 亏损上限 = Robobus 毛利
```

所以：

```text
情景 A：Robovan 最多亏 10,000,000
情景 B：Robovan 最多亏 12,500,000
```

## 5. 怎么用它反推 Robovan 能承受多少

下面两行“单车亏损”和“交付量”本质是在做反推。

### 平台模式

平台模式通常是：

```text
Robovan 总亏损 = 单车亏损 × 运营车辆数/交付量
```

因此：

```text
可承受车辆数 = Robovan 亏损上限 / 单车亏损
```

举例：

| 亏损上限 | 单车亏损 | 可承受车辆数 |
|---:|---:|---:|
| 10,000,000 | 10,000 | 1,000 |
| 10,000,000 | 5,000 | 2,000 |
| 12,500,000 | 10,000 | 1,250 |
| 12,500,000 | 5,000 | 2,500 |

### 卖车模式

卖车模式同理：

```text
Robovan 总亏损/毛利 = 单车亏损/毛利 × 交付量
```

如果卖车单车毛利是正的，那么它不是消耗毛利空间，而是增加毛利空间；如果卖车含返利后单车毛利很低甚至亏损，就需要看它会消耗多少 Robobus 毛利。

## 6. 这张表和前几张表怎么关联

前几张表算的是 Robovan 各模式的结果：

- 平台模式：全年毛利约 -777 万
- 租赁模式：全年毛利约 -672 万
- 卖车不含返利：全年毛利约 +6774 万
- 卖车含返利：全年毛利约 +1959 万

这张“毛利空间”表是在问：

> 如果 Robobus 一年能贡献 1000 万或 1250 万毛利，那么 Robovan 哪种模式的亏损/投入是可以被业务整体承受的？

按当前截图整理的粗略结果：

| Robovan 模式 | 全年毛利/亏损 | 是否在 1000万亏损上限内 |
|---|---:|---|
| 平台模式 | 约 -777万 | 可以覆盖 |
| 租赁模式 | 约 -672万 | 可以覆盖 |
| 卖车不含返利 | 约 +6774万 | 不消耗空间，反而贡献毛利 |
| 卖车含返利 | 约 +1959万 | 不消耗空间，仍贡献毛利 |

注意：这是基于截图人工整理的年度结果，后续要用原始表格精确重算。

## 7. 怎么给非财务同学解释

可以这样讲：

> Robobus 是已有业务，能产生一部分毛利；Robovan 是新业务，前期可能亏损。毛利空间表就是在问：已有业务一年赚的钱，最多能支持新业务亏多少？如果 Robovan 亏损小于这个空间，整体业务还能承受；如果超过，就需要更多资金或降低亏损。

更短：

```text
这张表不是算成本，
而是算“Robovan 可以亏到什么程度还不伤整体盘子”。
```

# 毛利空间 理解笔记

> 来源：用户“毛利空间”截图。当前为截图人工整理版，用于理解业务口径；若后续拿到原始表格，需要以原始表格为准补齐空白计算项。

## 1. 这张表在回答什么问题

这张表不是成本测算明细，而是在回答一个更上层的问题：

> 如果 Robobus 业务有一定毛利，最多能给 Robovan 业务承受多少亏损空间？

也可以理解为：

```text
Robobus 的毛利
→ 可以覆盖 Robovan 的亏损上限
→ 再反推 Robovan 平台模式/卖车模式的单车亏损和交付量空间
```

所以这张表更像是 **“毛利池 / 亏损承受空间”** 测算。

## 2. 表结构

截图中主要分为两部分：

```text
2026年

Robobus
- 收入
- 成本
- 毛利
- 毛利率

Robovan
- 亏损上限
- 平台模式
  - 单车亏损
  - 交付量
- 卖车模式
  - 单车亏损
  - 交付量
```

## 3. Robobus 部分

截图中给了两个情景列，可能是不同毛利率假设：

| 项目 | 情景 A | 情景 B |
|---|---:|---:|
| 收入 | 50,000,000 | 50,000,000 |
| 毛利 | 10,000,000 | 12,500,000 |
| 毛利率 | 20% | 25% |
| 成本 | 40,000,000（由收入-毛利反推） | 37,500,000（由收入-毛利反推） |

逻辑：

```text
Robobus 毛利 = 收入 × 毛利率
```

因此：

```text
50,000,000 × 20% = 10,000,000
50,000,000 × 25% = 12,500,000
```

## 4. Robovan 亏损上限

截图中 Robovan 的“亏损上限”直接对应 Robobus 的毛利：

| 项目 | 情景 A | 情景 B |
|---|---:|---:|
| Robobus 毛利 | 10,000,000 | 12,500,000 |
| Robovan 亏损上限 | 10,000,000 | 12,500,000 |

核心含义：

> 如果希望集团/整体业务不被 Robovan 拖累到亏损，那么 Robovan 的可承受亏损不能超过 Robobus 贡献的毛利。

也就是：

```text
Robovan 可亏损上限 = Robobus 可贡献毛利
```

## 5. 平台模式和卖车模式的反推逻辑

表中后面留了：

```text
平台模式：
- 单车亏损
- 交付量

卖车模式：
- 单车亏损
- 交付量
```

这部分应该用于做反推：

```text
可承受亏损上限 = 单车亏损 × 车辆/交付量
```

因此：

```text
可承受车辆/交付量 = 亏损上限 / 单车亏损
```

例如：

如果 Robovan 亏损上限是 10,000,000：

| 单车亏损 | 可承受数量 |
|---:|---:|
| 5,000 | 2,000 台 |
| 10,000 | 1,000 台 |
| 20,000 | 500 台 |

如果 Robovan 亏损上限是 12,500,000：

| 单车亏损 | 可承受数量 |
|---:|---:|
| 5,000 | 2,500 台 |
| 10,000 | 1,250 台 |
| 20,000 | 625 台 |

## 6. 怎么和前面三张成本表连接

这张表本身不直接算平台/租赁/卖车成本，它要引用前面成本测算表的结果：

### 平台模式

前面平台表可以给出：

```text
平台模式单车毛利 / 单车亏损
```

如果某月平台模式单车亏损是 498 元：

```text
1000万亏损空间 / 498 ≈ 20,080 台车
```

如果某个早期月份单车亏损是 15,883 元：

```text
1000万亏损空间 / 15,883 ≈ 629 台车
```

所以同样 1000 万亏损空间，在早期和后期能支撑的车辆规模差异非常大。

### 卖车模式

卖车模式可以引用：

```text
单车毛利 / 单车亏损
```

如果卖车含返利早期单车亏损约 35,475：

```text
1000万亏损空间 / 35,475 ≈ 282 台
```

如果后期卖车含返利单车毛利已经转正，则不再消耗亏损空间，而是反过来贡献毛利。

## 7. 这张表的业务意义

这张表是在做一个“承受能力”问题，而不是单一模式盈利性问题。

它回答：

1. Robobus 能贡献多少毛利？
2. 这部分毛利能容忍 Robovan 亏多少？
3. 如果 Robovan 单车还在亏，最多能支持多少车/多少交付？
4. 如果 Robovan 单车毛利转正，它是否能反过来贡献利润？

可以用一句话解释：

> Robobus 是毛利池，Robovan 是增长投入项。这张表用 Robobus 毛利去框定 Robovan 的亏损边界。

```

```

```text

```

|---|---:|---:|

### 4. 与三模式联动

把三种模式的单车亏损/毛利接进来：

```text

租赁模式当前单车亏损 x 元 → 可支撑 y 台
卖车模式当前单车毛利 x 元 → 不消耗亏损池/反向贡献
```

## 9. 对非财务同学怎么讲

可以这样讲：

> Robobus 能赚出来一笔毛利，比如 1000 万。Robovan 如果还在投入期，就不能无限亏。我们要看每辆车/每次交付亏多少钱，然后算这 1000 万最多能撑多少车、撑多久。

更短：

```text
这张表不是算某个模式赚不赚钱，
而是算“我们最多亏得起多少”。
```
