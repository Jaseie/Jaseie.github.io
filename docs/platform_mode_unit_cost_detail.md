# 平台模式：100台车运营预算与单车月度成本理解笔记

> 来源：用户“平台模式”截图。当前为截图人工整理版，用于理解成本口径；若后续拿到原始表格，需要以原表公式为准校验。

## 1. 这张表在回答什么问题

这张表不是全年 UE 表，而是一个更细的**平台模式单车月度成本底稿**。

它在回答：

> 假设 100 台车运营，每台车每月需要承担哪些固定成本、变动成本、运营成本？最后每台车每月收入 3000 元时，是赚还是亏？

表头可理解为：

```text
100台车运行预算，每台车每月成本
```

核心结论在底部：

```text
每月单车成本合计：5,278
每月单车收入：3,000
每月单车亏损：-2,278
```

## 2. 表的总体结构

这张表可以分为三段：

```text
每月固定成本
每车月变动成本
固定运营月成本
```

最后汇总成：

```text
每月单车成本合计
= 每月固定成本
+ 每车月变动成本
+ 固定运营月成本
```

## 3. 核心数据

| 成本模块 | 单车每月成本 |
|---|---:|
| 每月固定成本 | 2,747 |
| 每车月变动成本 | 2,292 |
| 固定运营月成本 | 240 |
| 每月单车成本合计 | 5,278 |
| 每月单车收入 | 3,000 |
| 每月单车亏损 | -2,278 |

最重要的公式：

```text
每月单车亏损 = 每月单车收入 - 每月单车成本合计
             = 3,000 - 5,278
             = -2,278
```

## 4. 每月固定成本

截图中“每月固定成本”为：

```text
2,747
```

其中主要是：

| 项目 | 单车每月成本 | 计算口径 |
|---|---:|---|
| 租金 | 2,747 | 整车成本 × 1.65 / 12 |

这里的“租金”更像车辆资产相关的月度成本或车辆租赁/资金成本摊销。

截图备注中可读到：

```text
整车成本 × 1.65 / 12
```

含义可以解释为：

> 单车资产成本按一定系数折算成每月租金/资产占用成本。

## 5. 每车月变动成本

截图中“每车月变动成本”为：

```text
2,292
```

明细如下：

| 成本项 | 单车每月成本 | 计算口径/备注 |
|---|---:|---|
| 充电 | 387 | 不持有车辆则不计算；截图备注提到目前约 29 台车 3 个月动力电 33668.84 |
| 运营安全员 | 209 | 人力成本 / 人车比 |
| 客服 | 50 | 人力成本 / 人车比；截图变量处有 1:60、AI客服 |
| 保险 | 105 | - |
| 流量 | 240 | 流量卡约 1440 元（500G）/ 年 |
| 云服务费用 | 180 | 相机个数/分钟率/工单/人车比等相关 |
| 补充运力 | 500 | 事故数量 × 500 元 |
| 采图制图 | 621 | 百度制图相关，截图备注约目前 2 台 18000 元 |

经营含义：

> 变动成本里最大的是采图制图、补充运力、充电、流量、运营安全员。这些都和运营效率、事故、地图/路线、车辆使用强度有关。

## 6. 固定运营月成本

截图中“固定运营月成本”为：

```text
240
```

明细如下：

| 成本项 | 单车每月成本 | 计算口径/备注 |
|---|---:|---|
| 仓储 | 50 | 不持有车辆则不计算；备注提到 10 元/平米/月、每台小车占用面积约 5 平米 |
| 地勤人员 | 50 | 人力成本 / 人车比；变量处约 1:50 |

经营含义：

> 固定运营月成本相对小，但会随着城市、线路、仓储和地勤配置变化。

## 7. 怎么看这张表的业务意义

这张表直接说明：

```text
平台模式在当前100台车运营预算下，
单车月收入 3000，
但单车月成本 5278，
所以每车每月亏 2278。
```

也就是说，平台模式要打平，需要做到至少其中一个：

1. **提高单车月收入**
2. **降低资产/租金成本**
3. **降低变动成本**
4. **摊薄固定运营成本**

## 8. 哪些成本最值得优先看

从单车成本贡献看：

| 成本项 | 单车月成本 | 占总成本比例 |
|---|---:|---:|
| 租金 | 2,747 | 约 52% |
| 采图制图 | 621 | 约 12% |
| 补充运力 | 500 | 约 9% |
| 充电 | 387 | 约 7% |
| 流量 | 240 | 约 5% |
| 运营安全员 | 209 | 约 4% |
| 云服务费用 | 180 | 约 3% |
| 其他 | - | - |

结论：

> 平台模式单车亏损的第一大来源是资产/租金成本，其次是采图制图、补充运力、充电、流量等运营项。

## 9. 和年度平台模式表怎么关联

前面“平台模式成本测算（含税）”是年度/月度 UE 汇总表。  
这张“平台模式”更像底层单车月成本底稿。

关系可以理解为：

```text
平台模式底稿
→ 得出单车每月各项成本
→ 汇总进入平台年度测算表
→ 形成每月成本、毛利、毛利率
```

其中：

- 租金/资产成本 → 年度表里的资产/租赁成本
- 充电、远驾/安全员、客服、保险、流量、云服务、补充运力、采图制图 → 年度表里的运营变动成本
- 仓储、地勤、路线部署 → 年度表里的运营固定成本

## 10. 给非财务同学怎么解释

可以这样讲：

> 这张表是在算一辆车一个月跑平台模式要花多少钱。现在一辆车每月收入 3000 元，但总成本 5278 元，所以每辆车每月亏 2278 元。要让平台模式打平，要么收入提升，要么租金/资产成本下降，要么把采图、补充运力、充电、流量等成本压下来。

更短：

```text
一辆车每月赚 3000，
但现在要花 5278，
所以差 2278。
```

## 11. 适合做成哪些可视化

这张表非常适合做成：

1. **单车月成本瀑布图**

```text
收入 3000
- 租金 2747
- 变动成本 2292
- 固定运营 240
= 单车亏损 -2278
```

2. **成本占比环形/条形图**

展示租金、采图、补充运力、充电、流量等各占多少。

3. **打平反推器**

```text
当前单车收入 3000
打平所需收入 5278
收入缺口 2278
```

或：

```text
如果收入不变，需要成本下降 43.2%
```

4. **优化杠杆排序**

把成本按金额排序，优先看：

```text
租金 > 采图制图 > 补充运力 > 充电 > 流量
```

## 12. 当前截图需要后续确认的地方

- “租金”是否等同车辆租赁/资产资金成本，需要确认。
- “整车成本 × 1.65 / 12”中的 1.65 是什么系数，需要业务解释。
- 充电 387 的具体计算口径需要原表确认。
- 云服务费用 180 的公式较复杂，需要原表确认。
- SD地图订阅费是否为 0 或未填，需要确认。
- 这张表是 100台车预算，不一定等同所有月份的年度模型，需要确认如何映射。
