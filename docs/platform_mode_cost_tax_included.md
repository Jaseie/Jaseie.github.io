# 平台模式成本测算（含税）理解笔记

> 来源：用户 2026 年平台模式成本测算截图。当前为截图人工整理版，用于先理解口径和结构；若后续拿到 Excel/CSV/飞书导出，需要以原始表格为准替换精确数值。

## 1. 这张表在回答什么问题

这张表是在算：**平台模式下，每个月运营一定数量的车，收入、成本、毛利分别是多少；成本又由哪些业务变量驱动。**

它不是单纯财务报表，而是一个经营测算模型。上半部分是结果，下半部分是原因：

- 上半部分：车辆数、收入、成本、毛利、毛利率、单车收入、单车成本、单车毛利。
- 中间部分：成本拆成三块：资产/租赁成本、运营变动成本、运营固定成本。
- 下半部分：每个成本项背后的参数，例如远驾人车比、MPRI、事故次数、硬件成本、IRR、能耗、维修等。

## 2. 最重要的阅读顺序

建议不要从第一行读到最后一行，而是按这 4 步看：

### 第一步：先看经营结果

核心行：

- 累计运营车辆数
- 收入
- 成本
- 毛利
- 毛利率
- 单车收入
- 单车成本
- 单车毛利

### 第二步：看亏损是不是随规模改善

这张表有一个非常重要的趋势：**单车亏损随着车辆规模扩大快速收窄。**

- 5月：单车毛利约 -15,883
- 6月：约 -9,837
- 7月：约 -3,658
- 10月：约 -780
- 11月：约 -498
- 12月：约 +55

直观理解：

> 平台模式早期很亏，不一定是单车收入太低，而是固定/半固定成本、人员成本、资产成本还没有被规模摊薄。随着车辆数从 100 增到 2300，单车成本被摊薄，12 月接近盈亏平衡。

### 第三步：拆成本结构

表里成本大致分成三层：

```text
单车成本
= 单位租赁/资产成本
+ 运营单位变动成本
+ 运营固定成本
```

截图中的关键行：

| 成本项 | 含义 | 2026年5月 | 2026年12月 | 经营含义 |
|---|---|---:|---:|---|
| 单位租赁/资产成本 | 车辆资产相关成本 | 1,823 | 1,148 | 硬件成本下降后，资产摊销下降 |
| 运营单位变动成本 | 会随运营效率变化的成本 | 16,481 | 1,497 | 规模扩大 + 人效改善后下降最明显 |
| 运营固定成本 | 仓储、地勤、路线部署等 | 300 | 300 | 单车固定口径在表里保持稳定 |
| 单车成本 | 三者相加 | 18,583/18,883 左右 | 2,945 | 决定单车毛利是否转正 |

最值得关注的是：**运营单位变动成本从 5 月的约 16,481 降到 12 月的约 1,497，是平台模式改善的主因。**

### 第四步：看变动成本的驱动参数

运营变动成本继续拆开：

```text
运营单位变动成本
= 充电
+ 远驾安全员
+ 客服
+ 保险
+ 风险赔偿
+ 流量
+ 云服务费用
+ 补充运力
+ 维修
+ 采图制图
```

几个最关键的业务驱动：

| 驱动项 | 表中参数 | 怎么影响成本 |
|---|---|---|
| 远驾人效 | 远驾人车比：1 → 13 | 人车比越高，单车远驾安全员成本越低 |
| 事故风险 | 百台车月事故次数、月事故次数、单次赔偿金额 | 影响风险赔偿、补充运力、维修等 |
| 云服务 | MPRI、工单时长、带宽、流量单价 | MPRI/工单越高，云服务成本越高 |
| 能耗 | 能耗、电费单价、空跑里程 | 影响充电成本 |
| 硬件成本 | 103,165 → 65,000 | 影响资产/租赁成本 |
| 规模 | 车辆数 100 → 2300 | 摊薄人员、运维、云服务等成本 |

## 3. 这张表可以怎么给非财务同学解释

可以用一句话解释：

> 平台模式的经营结果主要看三个动作：车能不能跑起来、运营效率能不能上去、事故和云服务成本能不能压住。收入单车每月 3000 元基本不变，真正决定毛利的是单车成本能不能从 1.8 万降到 3000 以下。

更白话一点：

1. 每辆车每月大约赚 3000 元收入。
2. 5 月每辆车成本接近 1.9 万，所以每辆亏 1.6 万。
3. 12 月车辆规模上来、硬件成本下降、人效变好后，每辆车成本降到约 2945 元。
4. 所以 12 月开始接近打平，甚至单车小幅转正。

## 4. 哪些岗位/动作会影响这张表

不按岗位做产品也可以，但理解时可以按业务动作看：

| 业务动作 | 影响的表格行 | 最终影响 |
|---|---|---|
| 提升远驾人车比 | 远驾安全员成本 | 降低运营变动成本，提高毛利 |
| 降低事故次数 | 风险赔偿、补充运力、维修 | 降低运营变动成本，提高毛利 |
| 降低硬件 BOM | 硬件成本、资产/租赁成本 | 降低单车成本，提高毛利 |
| 降低 MPRI/工单时长 | 云服务费用 | 降低运营变动成本 |
| 扩大车辆规模 | 车辆数、收入、成本摊薄 | 总收入上升，单车成本下降 |
| 降低能耗/空跑 | 充电成本 | 降低运营变动成本 |

## 5. 这张表最适合做哪些可视化

针对平台模式，最重要的图不是很多柱状图，而是下面 5 类：

1. **单车毛利趋势线**：看什么时候从亏损到打平。
2. **单车成本结构堆叠图**：资产成本、运营变动、运营固定分别占多少。
3. **运营变动成本拆解瀑布图**：远驾、风险、云服务、维修谁贡献最大。
4. **参数-指标因果树**：事故次数 → 风险赔偿/补充运力/维修 → 运营变动成本 → 单车毛利。
5. **敏感性图**：事故、硬件、MPRI、远驾人车比等各自变化 10%，对毛利影响多大。
